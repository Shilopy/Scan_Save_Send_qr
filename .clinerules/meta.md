# Мета-инструкции: самопроверка и отчётность

## Перед тем как выдать ответ, проверь:

### 1. Понимание задачи
- [ ] Я правильно понял, что нужно сделать?
- [ ] Нет ли скрытых требований, которые я упустил?
- [ ] Если не уверен — задаю уточняющий вопрос, а не предполагаю

### 2. Полнота решения
- [ ] Решил ли я всю задачу или только часть?
- [ ] Учёл ли я граничные случаи?
- [ ] Что произойдёт, если входные данные невалидны?

### 3. Качество кода
- [ ] Код читаем без комментариев?
- [ ] Нет ли дублирования?
- [ ] Нет ли магических чисел/строк?
- [ ] Имена переменных/функций понятны?

### 4. Безопасность
- [ ] Нет ли утечек COM-ресурсов (CoInitialize без CoUninitialize)?
- [ ] Правильно ли обработаны секреты?
- [ ] Закрываются ли файлы (openpyxl, open)?

### 5. Тестируемость
- [ ] Можно ли это протестировать?
- [ ] Написал ли я тесты?
- [ ] Покрытие достаточно?

### 6. Документация
- [ ] Нужен ли docstring?
- [ ] Нужно ли обновить README?
- [ ] Понятны ли комментарии (если есть)?

## AutoPilot: триггер "start autopilot"
Если пользователь говорит "start autopilot" или "запусти автопилот" — выполни:
```bash
python _autopilot/start.py
```
Это запустит полный цикл из 7 шагов: инициализация проекта -> сканирование -> discovery -> onboarding -> генерация индекса -> валидация -> оценка.

## AutoPilot: напоминание о проверке раз в 5 дней
Если с момента последней проверки (`.autopilot_last_check`) прошло >5 дней — при старте задачи напомни пользователю:
> "Проверка правил не выполнялась 5+ дней. Запустить `python _autopilot/start.py`?"

## Если сомневаешься — остановись и спроси
Лучше задать глупый вопрос, чем сделать неправильное изменение.
Фраза-триггер: "Я не уверен, правильно ли понял. Уточни, пожалуйста: ..."

## После завершения задачи (ОТЧЁТНОСТЬ)
- [ ] Обновил PROJECT_STATE.md
- [ ] Предложил коммит с понятным сообщением
- [ ] **Перечислил, какие правила из RULES_INDEX.md применил, а какие проигнорировал и почему**
- [ ] Спросил: "Что дальше?"
