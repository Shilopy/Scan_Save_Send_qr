# Обработка ошибок в AutoSender

## Что
Все ошибки должны логироваться через `logger.error()` с полным контекстом (функция, входные данные). Критические ошибки — через `logger.critical()` и глобальный `sys.excepthook`.

## Когда применять
- При ошибке подключения к Outlook (COMException, таймаут)
- При ошибке отправки письма (SMTP, COM, таймаут)
- При ошибке чтения Excel-файла (openpyxl)
- При ошибке доступа к папке или файлам (OSError, PermissionError)
- При ошибке загрузки/сохранения конфигурации (JSON)
- При любой неожиданной ошибке в цикле мониторинга

## Когда НЕ применять
- Для ошибок валидации ввода пользователя в Streamlit UI (используй `st.error()`)
- Для debug-сообщений (используй `logger.debug()`)

## Почему
- Сервис работает без присмотра 24/7 — ошибки должны быть видны в логах без открытия UI
- Таймауты Outlook (30-120 сек) не должны блокировать UI или цикл мониторинга
- Глобальный `sys.excepthook` ловит то, что не поймано нигде — последняя линия обороны

## Пример (правильно)
```python
try:
    outlook = win32com.client.Dispatch("Outlook.Application")
except Exception as e:
    logger.error(f"Ошибка подключения к Outlook: {e}")
    st.error(f"Ошибка подключения к Outlook: {e}")
    return None
```

## Антипаттерн (неправильно)
```python
# Плохо: ошибка проглатывается без контекста
try:
    send_email(...)
except:
    pass
```
