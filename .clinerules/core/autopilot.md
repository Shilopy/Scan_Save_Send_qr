# AutoPilot: самоорганизующаяся система управления .clinerules

## Что
AutoPilot — это система скриптов в `_autopilot/`, которая полностью автоматизирует управление правилами `.clinerules`. Пользователь говорит "start autopilot" — агент делает всё остальное: от инициализации проекта до оценки правил.

## Когда применять
- **При старте нового проекта** — пользователь говорит "start autopilot", AutoPilot сам определяет стек, создаёт .clinerules, инициализирует Git, делает первый коммит
- Раз в 5 дней (напоминание о проверке)
- После добавления новых правил вручную
- При переносе `.clinerules` в новый проект

## Когда НЕ применять
- Если проект не использует `.clinerules`
- Если пользователь явно отключил автопилот

## Почему
Ручное управление правилами отнимает время и приводит к рассинхронизации. AutoPilot обеспечивает актуальность, целостность и релевантность правил без участия пользователя.

## Команды

| Команда | Описание |
|---|---|
| `start autopilot` | **Триггер для Cline**: запускает полный цикл (см. ниже) |
| `python _autopilot/start.py` | Полный цикл (7 шагов) |
| `python _autopilot/start.py --auto` | Авто-режим (без вопросов) |
| `python _autopilot/start.py --skip-init` | Пропустить инициализацию проекта |
| `python _autopilot/start.py --skip-discover` | Пропустить поиск новых правил |
| `python _autopilot/start.py --skip-evaluate` | Пропустить оценку |
| `python _autopilot/detect_stack.py` | Только определение стека проекта |
| `python _autopilot/init_project.py` | Только инициализация (.clinerules + Git + коммит) |
| `python _autopilot/discover_rules.py` | Только поиск новых правил на GitHub |
| `python _autopilot/onboard_rules.py --rules FILE` | Только интеграция правил из JSON |
| `python _autopilot/auto_heal.py` | Только автоисправление ошибок |
| `python _autopilot/evaluate_rules.py` | Только оценка использования |

## Полный цикл (7 шагов)

```
[1/7] Инициализация проекта
  ├── Определение стека (язык, фреймворк, БД)
  ├── Создание .clinerules/ (если нет)
  ├── Заполнение base.md стеком
  ├── Инициализация Git (если нет)
  ├── Создание .gitignore
  └── Первый коммит

[2/7] Сканирование проекта
  ├── Проверка .clinerules, tools/, Git
  └── Чтение стека из base.md

[3/7] Discovery — поиск новых правил на GitHub
  ├── Поиск под стек проекта
  └── Фильтрация безопасных/опасных

[4/7] Onboarding — интеграция правил
  ├── Копирование в .clinerules/discovered/
  ├── Переформатирование в Compact/Full
  └── Обновление индекса

[5/7] Генерация RULES_INDEX.md

[6/7] Валидация + автоисправление
  ├── Проверка целостности
  └── Автоисправление ошибок

[7/7] Оценка использования правил
  └── Какие правила применялись, какие устарели
```

## Пример (правильно)

```python
# Пользователь говорит Cline: "start autopilot"
# Cline запускает: python _autopilot/start.py

# AutoPilot:
# [1/7] Определяет стек (Python, Streamlit, SQLite)
#       Создаёт .clinerules/ с base.md, meta.md, core/, modes/, templates/
#       Инициализирует Git, создаёт .gitignore, первый коммит
# [2/7] Сканирует проект — 5 правил, Git есть
# [3/7] Ищет на GitHub правила под Python/Streamlit
#       Найдено 3 новых правила
# [4/7] Пользователь: y — интеграция
# [5/7] RULES_INDEX.md обновлён
# [6/7] Валидация: OK
# [7/7] Оценка: 2 правила используются, 1 устарело
```

## Антипаттерн (неправильно)

```python
# Пользователь вручную:
# 1. Создаёт .clinerules/base.md
# 2. Пишет стек
# 3. Ищет правила на GitHub
# 4. Копирует, форматирует
# 5. Запускает индексацию
# 6. Запускает валидацию
# 7. Исправляет ошибки
# Всё это делает AutoPilot за 1 команду
```

## Связанные правила
- `meta.md` — напоминание о проверке раз в 5 дней, триггер "start autopilot"
- `core/error-handling.md` — обработка ошибок в скриптах
- `core/logging.md` — логирование работы автопилота
