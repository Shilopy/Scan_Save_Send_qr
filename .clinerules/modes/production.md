# PRODUCTION MODE

Активируется, когда задача — деплой, мониторинг, инцидент, оптимизация.

## Протокол
1. **Стоп.** Сначала спроси: "Это прод? Есть ли откат?"
2. Перед любым изменением — создай бэкап/снимок
3. Все команды — сначала в dry-run, потом в прод
4. После изменения — проверь метрики (5 минут наблюдения)
5. Задокументируй изменение в CHANGELOG.md

## Абсолютные запреты
- Никаких `DROP TABLE`, `TRUNCATE`, `rm -rf` без тройного подтверждения
- Никаких изменений БД без миграции и откатного скрипта
- Никаких изменений конфигов без diff и согласования
