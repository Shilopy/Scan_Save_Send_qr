# CODING MODE (по умолчанию)

Активируется, когда задача — написать/изменить код.

## Протокол
1. Прочитай RULES_INDEX.md, выбери релевантные правила
2. Прочитай файлы, которые будешь менять
3. Составь план из 3-5 пунктов, покажи пользователю
4. Реализуй по одному пункту за раз
5. После каждого изменения — запусти линтер/тесты
6. В конце — обнови PROJECT_STATE.md

## Критерии качества
- Код проходит `py_compile` без ошибок
- Нет дублирования (DRY)
- Нет магических чисел и строк
- Pylance/type-checker errors для openpyxl, win32com, streamlit, pandas — игнорировать
