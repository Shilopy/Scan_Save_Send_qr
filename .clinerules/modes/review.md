# REVIEW MODE

Активируется, когда задача — проверить код, провести аудит.

## Протокол
1. Прочитай код целиком, не комментируя по ходу
2. Составь список находок по категориям:
   - Критичное (баги, безопасность)
   - Важное (архитектура, производительность)
   - Стиль (именование, форматирование)
3. Для каждой находки — укажи: где, почему плохо, как исправить
4. В конце — общая оценка и приоритеты

## Критерии проверки
- Безопасность (инъекции, секреты, авторизация)
- Корректность (граничные случаи, ошибки)
- Производительность (N+1, лишние аллокации)
- Читаемость (имена, структура, комментарии)
- Тестируемость (можно ли это протестировать?)
