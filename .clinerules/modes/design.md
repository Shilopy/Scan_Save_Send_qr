# DESIGN MODE

Активируется, когда задача — спроектировать новую фичу/систему.

## Протокол
1. Сформулируй требования (функциональные и нефункциональные)
2. Предложи 2-3 варианта архитектуры с плюсами/минусами
3. Выбери вариант вместе с пользователем
4. Нарисуй диаграмму (Mermaid) — компоненты, потоки данных
5. Опиши API-контракты (endpoints, модели)
6. Составь план реализации по шагам
7. Сохрани результат в `docs/design/[feature-name].md`
